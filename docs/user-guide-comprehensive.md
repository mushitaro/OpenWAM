# OpenWAM エンジンシミュレーター - 包括的ユーザーガイド

## 概要

OpenWAM エンジンシミュレーター Webアプリケーションは、エンジンシミュレーションを直感的なWebインターフェースで実行できる包括的なツールです。ドラッグ&ドロップによるビジュアルモデリング、リアルタイムシミュレーション実行、インタラクティブな結果可視化を提供します。

## はじめに

### システム要件

#### 必須要件

- **Node.js**: バージョン18以上
- **npm**: Node.jsに付属（またはyarn）
- **ブラウザ**: Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
- **OS**: Windows 10+, macOS 10.15+, Ubuntu 18.04+

#### 推奨要件

- **画面解像度**: 1280x720以上（1920x1080推奨）
- **メモリ**: 4GB以上（8GB推奨）
- **ストレージ**: 2GB以上の空き容量
- **CPU**: デュアルコア以上（クアッドコア推奨）

#### OpenWAM実行ファイル

- **OpenWAM実行ファイル**: シミュレーション実行に必要
- **対応バージョン**: OpenWAM 2200以上
- **配置場所**: `bin/debug/OpenWAM` または環境変数で指定

### インストールと初期設定

#### 1. リポジトリのクローンと依存関係インストール

```bash
# リポジトリをクローン
git clone <repository-url>
cd openwam-engine-simulator

# サーバー依存関係をインストール
npm install

# クライアント依存関係をインストール
cd client
npm install
cd ..
```

#### 2. OpenWAMのビルド（必要な場合）

```bash
# ビルドディレクトリを作成
mkdir build && cd build

# CMakeでビルド設定
cmake ..

# 並列サポート付きビルド（推奨）
cmake -DBUILD_PARALLEL=ON ..

# ビルド実行
make

# 実行権限を付与
chmod +x bin/debug/OpenWAM
```

#### 3. アプリケーションの起動

```bash
# 開発モード（推奨）
npm run dev

# または本番モード
npm run build
npm start
```

#### 4. ブラウザでアクセス

ブラウザで `http://localhost:3001` にアクセスしてアプリケーションを開始します。

### 初回起動時の設定

1. **システムステータス確認**: ダッシュボードでシステム状態を確認
2. **OpenWAM実行ファイル確認**: シミュレーション機能が利用可能か確認
3. **サンプルプロジェクト**: 提供されるサンプルで動作確認

## ビジュアルモデルエディター

### インターフェース概要

ビジュアルモデルエディターは以下の要素で構成されています：

1. **コンポーネントパレット**（左側）
2. **メインキャンバス**（中央）
3. **プロパティパネル**（右側）
4. **ツールバー**（上部）
5. **ステータスバー**（下部）

### コンポーネントパレット

#### カテゴリ構成

**パイプ要素**
- 1Dパイプ (TTubo)
- 同心円パイプ (TConcentrico)

**境界条件**
- 開放端（大気）
- 閉端
- 無反射端
- 分岐

**プレナム要素**
- 定容積プレナム
- 可変容積プレナム
- シンプルタービン

**バルブ要素**
- 固定CDバルブ
- 4Tバルブ
- リードバルブ
- バタフライバルブ

**エンジン要素**
- エンジンブロック
- 4Tシリンダー
- 2Tシリンダー

**DPF要素**
- ディーゼル微粒子フィルター

#### コンポーネントの追加方法

**方法1: ドラッグ&ドロップ**
1. パレットからコンポーネントを選択
2. キャンバスにドラッグ
3. 適切な位置でドロップ

**方法2: クリック配置**
1. パレットでコンポーネントをクリック
2. キャンバス上の配置位置をクリック
3. コンポーネントが自動配置

### キャンバス操作

#### 基本操作

**ナビゲーション**
- **パン**: マウス中ボタンドラッグまたはSpaceキー+ドラッグ
- **ズーム**: マウスホイールまたはCtrl+マウスホイール
- **フィット**: Fキーで全体表示

**選択操作**
- **単一選択**: コンポーネントをクリック
- **複数選択**: Ctrlキー+クリック
- **範囲選択**: ドラッグで矩形選択

**編集操作**
- **移動**: 選択後ドラッグ
- **回転**: Rキー+マウス移動
- **削除**: Deleteキーまたは右クリックメニュー
- **複製**: Ctrl+D

#### 接続の作成

**手動接続**
1. 接続モードを有効化（Cキー）
2. 開始コンポーネントの接続ポイントをクリック
3. 終了コンポーネントの接続ポイントをクリック
4. 接続が自動作成

**自動接続**
- 互換性のあるコンポーネントを近づけると自動接続候補を表示
- 緑色のハイライトで有効な接続を示す
- 赤色のハイライトで無効な接続を警告

#### グリッドとスナップ

**グリッド設定**
- グリッド表示のON/OFF
- グリッドサイズの調整
- グリッドスナップの有効化

**スナップ機能**
- コンポーネント間のスナップ
- 接続ポイントへのスナップ
- 角度スナップ（15度単位）

### プロパティパネル

#### 基本情報

選択されたコンポーネントの基本情報を表示：
- コンポーネントタイプ
- 一意識別子

- 説明（編集可能）

#### パラメータ設定

コンポーネントタイプに応じたパラメータを表示：

**入力フィールドタイプ**
- **数値入力**: 単位付き数値フィールド
- **選択リスト**: プリセット値からの選択
- **チェックボックス**: ON/OFF設定
- **テキスト入力**: 自由テキスト

**検証機能**
- リアルタイム入力検証
- 範囲チェック
- 必須フィールドの確認
- 単位変換サポート

#### ヘルプとガイダンス

- パラメータ説明のツールチップ
- 推奨値の表示
- 関連パラメータの依存関係表示
- エラーメッセージと修正提案

## コンポーネント詳細ガイド

### パイプ要素

#### 1Dパイプ (TTubo)

**概要**: 1次元ガス流動計算の基本要素

**主要パラメータ**:

| パラメータ | 単位 | 説明 | 推奨範囲 |
|-----------|------|------|----------|
| 長さ | m | パイプ全長 | 0.1-5.0 |
| 直径 | m | パイプ内径 | 0.02-0.2 |
| セル数 | - | 計算セル数 | 10-100 |
| 摩擦係数 | - | 摩擦損失係数 | 0.01-0.05 |
| 壁厚 | m | 壁厚さ | 0.001-0.01 |

**設定のコツ**:
- セル数は長さ1mあたり10-20個が目安
- 摩擦係数は管内面粗さに応じて調整
- 高周波現象を捉える場合はセル数を増加

**適用例**:
- 吸気管・排気管
- インタークーラー配管
- EGR配管

#### 同心円パイプ (TConcentrico)

**概要**: 内管と外管を持つ二重管構造

**用途**:
- EGRクーラー
- インタークーラー
- 熱交換器

**特殊パラメータ**:
- 内管・外管の幾何パラメータ
- 熱交換係数
- 冷却媒体条件

### 境界条件

#### 開放端（大気）

**物理的意味**: パイプ端部が大気に開放
**反射特性**: 部分反射（周波数依存）
**用途**: エアクリーナー入口、マフラー出口

**設定パラメータ**:
- 基準圧力（通常は大気圧）
- 基準温度
- 流量係数

#### 閉端

**物理的意味**: 完全に閉じられた端部
**反射特性**: 完全反射
**用途**: 行き止まり配管、測定用配管端部

#### 無反射端

**物理的意味**: 理想的な無反射境界
**反射特性**: 反射なし
**用途**: 計算領域の切り取り、無限長配管の模擬

#### 分岐

**物理的意味**: 1本から2本への分岐または合流
**設定項目**:
- 各パイプの直径
- 分岐角度
- 損失係数

**設計指針**:
- 急激な断面変化を避ける
- 分岐角度は30度以下が望ましい

### プレナム要素

#### 定容積プレナム

**物理的意味**: 一定容積のチャンバー
**効果**: 圧力変動の平滑化、容積効果

**主要パラメータ**:
- 容積: エンジン排気量の10-50%が目安
- 初期圧力・温度
- 壁面熱伝達条件

**適用例**:
- エアクリーナーボックス
- サージタンク
- インテークマニホールド

#### 可変容積プレナム

**物理的意味**: 時間変化する容積
**容積変化関数**:
- 正弦波変化
- テーブル補間
- カスタム関数

**用途**:
- 可変吸気システム
- アクティブ制振システム

#### シンプルタービン

**物理的意味**: ターボチャージャータービン
**モデル**: マップベース性能計算

**設定項目**:
- タービンマップデータ
- 効率特性
- 慣性モーメント

### バルブ要素

#### 固定CDバルブ

**物理的意味**: 固定流量係数のオリフィス
**流量計算**: CD値による流量制限

**設定パラメータ**:
- 流量係数（CD値）: 0.3-0.8
- 有効面積
- 流れ方向

#### 4Tバルブ

**物理的意味**: 4ストロークエンジン用バルブ
**駆動方式**: カムプロファイルによる開閉

**主要設定**:
- バルブ径
- 最大リフト
- 開閉タイミング（クランク角度）
- リフトカーブ

**タイミング設定例**:
```
吸気バルブ:
- 開弁: 上死点前20度
- 閉弁: 下死点後60度

排気バルブ:
- 開弁: 下死点前50度
- 閉弁: 上死点後10度
```

#### リードバルブ

**物理的意味**: 圧力差駆動の自動バルブ
**用途**: 2ストロークエンジン吸気

**設定項目**:
- 開弁圧力差
- 有効面積
- バネ定数
- 応答特性

#### バタフライバルブ

**物理的意味**: 回転式スロットルバルブ
**用途**: 吸気量制御

**特性**:
- 開度と流量の非線形関係
- 小開度での高感度
- 全開付近での飽和特性

### エンジン要素

#### エンジンブロック

**役割**: エンジン基本諸元の定義

**主要パラメータ**:
- エンジンタイプ（2T/4T）
- 気筒数
- ボア径・ストローク
- コンロッド長
- 圧縮比
- 燃料タイプ

**設計指針**:
- ボア/ストローク比: 0.7-1.3が一般的
- コンロッド長: ストローク×1.5-2.0
- 圧縮比: ガソリン8-12、ディーゼル14-22

#### 4Tシリンダー

**物理的意味**: 4ストロークサイクルの実行

**主要設定**:
- バルブタイミング
- 燃料噴射量・時期
- 点火時期
- 燃焼モデルパラメータ

**バルブタイミング最適化**:
- 充填効率の向上
- 残留ガス率の制御
- ポンピング損失の低減

#### 2Tシリンダー

**物理的意味**: 2ストロークサイクルの実行

**特徴**:
- ポート制御ガス交換
- 掃気効率が重要
- 混合気損失の考慮

**主要設定**:
- ポートタイミング
- 掃気効率
- 燃料噴射設定

### DPF要素

#### ディーゼル微粒子フィルター

**物理的意味**: PM捕集フィルター

**主要パラメータ**:
- フィルター寸法（長さ・直径）
- セル密度（cpsi）
- 壁厚・多孔度
- 透過率・捕集効率

**モデル機能**:
- 圧力損失計算
- PM蓄積量計算
- 再生プロセス
- 温度分布

## シミュレーション実行

### 事前準備

#### モデル検証

**必須チェック項目**:
1. すべてのコンポーネントが接続されている
2. 必須パラメータがすべて設定されている
3. 物理的に妥当な値が設定されている
4. 循環参照がない

**検証ツール**:
- 自動検証機能
- 警告・エラー表示
- 推奨値との比較

#### シミュレーション設定

**基本設定**:
- 時間刻み: CFL条件を満足する値
- 総時間: エンジン数サイクル分
- 出力間隔: 解析目的に応じて設定

**高度な設定**:
- 収束判定条件
- 数値解法パラメータ
- 並列計算設定

### 実行プロセス

#### シミュレーション開始

1. **最終確認**
   - モデル検証結果の確認

   - 推定実行時間の確認

2. **実行開始**
   - 「シミュレーション開始」ボタンをクリック
   - OpenWAM入力ファイルの自動生成
   - バックグラウンドでOpenWAM実行

#### 進行状況監視

**リアルタイム表示**:
- 進行率（%）
- 経過時間
- 推定残り時間
- 現在の計算ステップ

**ログ監視**:
- OpenWAM出力メッセージ
- エラー・警告メッセージ
- 収束状況

#### 実行制御

**一時停止・再開**:
- 実行中の一時停止
- 一時停止からの再開
- チェックポイントからの再開

**強制終了**:
- 異常時の強制終了
- 部分結果の保存
- エラーログの保存

### トラブルシューティング

#### 一般的な問題

**実行開始しない**:
- OpenWAM実行ファイルの確認
- 入力ファイル生成エラーの確認
- ファイル権限の確認

**収束しない**:
- 時間刻みの調整
- 初期条件の見直し
- メッシュ密度の調整

**異常終了**:
- エラーログの確認
- メモリ不足の確認
- ディスク容量の確認
