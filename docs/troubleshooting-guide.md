## インストールと起動の問題

### Node.js バージョンの互換性問題

**症状**: アプリケーション起動時にNode.jsバージョンエラーが発生

**対処法**:
- Node.js バージョン 18 以上がインストールされていることを確認
- バージョン確認: `node --version`
- 必要に応じて [nodejs.org](https://nodejs.org) から更新

### 依存関係のインストール失敗

**症状**: `npm install` が権限エラーやネットワークエラーで失敗

**対処法**:
```bash
# npmキャッシュをクリア
npm cache clean --force

# node_modules と package-lock.json を削除
rm -rf node_modules package-lock.json
rm -rf client/node_modules client/package-lock.json

# 依存関係を再インストール
npm install
cd client && npm install
```

### OpenWAM コンパイル問題

**症状**: OpenWAM実行ファイルが見つからない、またはコンパイルが失敗

**対処法**:
```bash
# ソースからOpenWAMをビルド
mkdir build && cd build
cmake ..
make

# 並列ビルドの場合
cmake -DBUILD_PARALLEL=ON ..
make

# 実行ファイルの確認
ls -la bin/debug/OpenWAM
chmod +x bin/debug/OpenWAM
```

### サーバー起動失敗

**症状**: サーバーがポート3001で起動しない

**一般的な原因と対処法**:

1. **ポート使用中エラー**
   ```bash
   # ポート使用状況確認
   netstat -an | grep 3001
   
   # プロセス終了
   pkill -f "node.*server"
   
   # 別ポート使用
   PORT=3002 npm start
   ```

2. **ファイアウォール設定**
   - ローカルファイアウォールの設定確認
   - ポート3001の通信許可

3. **データベース接続問題**
   ```bash
   # ファイル権限確認
   chmod 755 app_data/
   
   # データベース再作成
   rm app_data/openwam.db
   ```

### クライアントビルド失敗

**症状**: React クライアントのビルドまたは起動が失敗

**対処法**:
```bash
# クライアントキャッシュクリア
cd client
rm -rf node_modules package-lock.json
npm install

# TypeScript エラー確認
npm run type-check

# クライアントビルド
npm run build
```

## コンポーネントとモデル作成の問題

### コンポーネントパレットが空または不完全

**症状**: 使用可能なコンポーネントが表示されない

**対処法**:
1. ブラウザの開発者ツールでネットワークリクエストを確認
2. コンポーネント定義APIエンドポイントの動作確認
3. サーバーログでエラーを確認
4. アプリケーションの再起動

### コンポーネント配置の問題

**症状**: コンポーネントをキャンバスにドラッグしても表示されない

**トラブルシューティング**:
1. ブラウザコンソールでJavaScriptエラーを確認
2. キャンバスの初期化状態を確認
3. ページの再読み込み
4. ブラウザキャッシュのクリア
5. コンポーネント定義の読み込み状況確認

### キャンバス描画問題

**症状**: キャンバスが表示されない、または空白画面

**対処法**:
1. ブラウザのWebGLサポート確認
2. グラフィックドライバーの更新
3. 別ブラウザでの動作確認
4. 必要に応じてハードウェアアクセラレーション無効化
5. キャンバスサイズとズームレベルの確認

### コンポーネント接続の問題

**症状**: コンポーネント同士を接続できない

**一般的な原因と対処法**:

1. **互換性のないコンポーネントタイプ**
   - ドキュメントで接続ルールを確認
   - 適切な境界条件を使用
   - ノードタイプ（入力/出力/双方向）を確認

2. **接続数制限**
   - ノードあたりの最大接続数を確認
   - 複数接続には分岐コンポーネントを使用
   - 既存接続の確認

3. **検証エラー**
   - コンポーネントプロパティが有効であることを確認
   - 循環参照の確認
   - ノード番号の確認

### コンポーネントプロパティ検証エラー

**症状**: コンポーネントプロパティで検証エラーが表示される

**コンポーネント別の一般的な問題と対処法**:

1. **パイプコンポーネント**:
   - 長さは正の値（> 0.001 m）
   - セル数は3-1000の範囲
   - 直径は現実的な値（0.01-0.5 m）
   - 摩擦係数は通常0.01-0.05

2. **エンジンコンポーネント**:
   - ボア/ストローク比は0.5-2.0
   - 圧縮比: 現実的なエンジンでは8-25
   - コンロッド長 > 1.5 × ストローク
   - バルブタイミング角度は±180°以内

3. **プレナムコンポーネント**:
   - 容積は正の値
   - 初期圧力 > 0.1 bar
   - 温度 > 273 K（0°C）

## シミュレーション実行の問題

### OpenWAM実行失敗

**症状**: シミュレーションが開始しない、または即座にクラッシュ

**詳細なトラブルシューティング**:

1. **OpenWAMインストール確認**:
   ```bash
   # 実行ファイルの存在と実行権限確認
   ls -la bin/debug/OpenWAM
   chmod +x bin/debug/OpenWAM
   
   # 基本実行テスト
   ./bin/debug/OpenWAM --version
   ```

2. **生成された入力ファイルの検証**:
   - .wamファイルの構文確認
   - コンポーネント番号の連続性確認
   - 必要なセクションの存在確認
   - 特殊文字やエンコーディング問題の確認

3. **一般的な入力ファイル問題**:
   - エンジンコンポーネント存在時のエンジンデータ不足
   - 不正なノード番号
   - 無効な境界条件タイプ
   - 一貫性のないコンポーネント参照

### シミュレーション収束問題

**症状**: シミュレーションが実行されるが収束しない、または不安定な結果

**診断と対処法**:

1. **時間刻み問題**:
   ```
   # CFL条件の確認
   CFL = (速度 × 時間刻み) / セルサイズ
   # 安定性のためCFL < 1であるべき
   ```
   - CFL > 1の場合は時間刻みを小さくする
   - メッシュ解像度を上げる
   - 超音速流れの確認

2. **初期条件**:
   - 現実的な初期圧力と温度を使用
   - 初期状態での質量保存を確保
   - 極端な勾配を避ける

3. **境界条件**:
   - 矛盾する境界条件の確認
   - 適切な圧力基準の確保
   - 流れ方向の確認

### シミュレーション性能問題

**症状**: シミュレーションが非常に遅い、または過度にメモリを使用

**最適化戦略**:

1. **メッシュ最適化**:
   - 可能な場合はパイプセル数を削減
   - 利用可能な場合は適応メッシュを使用
   - 精度と性能のバランス

2. **モデル簡略化**:
   - 不要なコンポーネントの削除
   - 簡略化された境界条件の使用
   - 出力頻度の削減

3. **並列処理**:
   ```bash
   # 並列サポートでコンパイルされている場合のOpenMP有効化
   export OMP_NUM_THREADS=4
   ```

### 結果解析問題

**症状**: 結果が物理的に非現実的

**検証チェックリスト**:

1. **質量保存**:
   - 入口と出口の質量流量確認
   - プレナムでの蓄積確認
   - 質量漏れの確認

2. **エネルギー保存**:
   - 温度変化の確認
   - 熱伝達計算の確認
   - エネルギー源/シンクの確認

3. **圧力波伝播**:
   - 波速の確認
   - 反射係数の確認
   - 数値分散の確認

## ファイル操作の問題

### ファイルアップロード/ダウンロード問題

**症状**: .wamファイルのアップロードまたは結果のダウンロードができない

**対処法**:
1. ファイルサイズ制限の確認（デフォルト: 50MB）
2. ファイル形式と拡張子の確認
3. ブラウザセキュリティ設定の確認
4. 適切なMIMEタイプの確保
5. より小さなファイルでのテスト

### ファイル形式問題

**症状**: アップロードされたファイルが認識されない、または正しく解析されない

**一般的な問題**:
1. **文字エンコーディング**: UTF-8エンコーディングを使用
2. **改行コード**: Unix形式の改行（\n）を使用
3. **ファイル構造**: OpenWAM形式に正確に従う
4. **数値形式**: カンマではなく小数点を使用

### プロジェクト保存/読み込み問題

**症状**: プロジェクトが正しく保存または読み込まれない

**トラブルシューティング**:
1. データベース接続の確認
2. JSONシリアライゼーションの確認
3. モデルデータの循環参照確認
4. 適切なエラーハンドリングの確保

## パフォーマンスの問題

### ブラウザパフォーマンス

**症状**: UIが遅くなる、または応答しなくなる

**最適化手順**:

1. **ブラウザ設定**:
   - ハードウェアアクセラレーション有効化
   - 不要なタブを閉じる
   - キャッシュとクッキーのクリア
   - 不要な拡張機能の無効化

2. **キャンバスパフォーマンス**:
   - ズームレベルの削減
   - 表示コンポーネントの制限
   - 簡略化描画モードの使用
   - 必要に応じてアニメーション無効化

3. **メモリ管理**:
   - 開発者ツールでメモリ使用量監視
   - 大きな結果データセットのクリア
   - 定期的なブラウザ再起動

### サーバーパフォーマンス

**症状**: サーバー応答が遅い

**対処法**:
1. **Node.js最適化**:
   ```bash
   # メモリ制限増加
   node --max-old-space-size=4096 src/server/server.js
   
   # V8最適化有効化
   node --optimize-for-size src/server/server.js
   ```

2. **データベース最適化**:
   ```sql
   -- SQLiteデータベースのバキューム
   VACUUM;
   
   -- クエリ性能解析
   EXPLAIN QUERY PLAN SELECT * FROM projects;
   ```

3. **ファイルシステム**:
   - より良いI/OのためSSDストレージを使用
   - 一時ファイルの定期的なクリーンアップ
   - ディスク容量の監視

## OpenWAM固有の問題

### 入力ファイル生成問題

**症状**: 生成された.wamファイルがOpenWAMエラーを引き起こす

**一般的な問題と修正**:

1. **セクション順序**: セクションが正しい順序で表示されることを確認
2. **データ形式**: 適切な数値形式を使用
3. **必須フィールド**: すべての必須パラメータを含める
4. **一貫性**: コンポーネント参照が有効であることを確認

### OpenWAMエラーコード

**一般的なエラーコードと対処法**:

| エラーコード | 意味 | 対処法 |
|------------|------|--------|
| 1 | 入力ファイルエラー | ファイル構文と形式を確認 |
| 2 | メモリ割り当て | モデルサイズを削減またはメモリ増加 |
| 3 | 収束失敗 | 時間刻みまたは初期条件を調整 |
| 4 | ファイルI/Oエラー | ファイル権限とディスク容量を確認 |

### バージョン互換性

**症状**: OpenWAMバージョンの非互換性

**対処法**:
1. OpenWAMバージョン要件の確認
2. 必要に応じて入力ファイル形式の更新
3. 互換性のある機能セットの使用
4. 参照ケースでのテスト

## エラーメッセージ一覧

### アプリケーションエラー

| エラーメッセージ | 原因 | 対処法 |
|-----------------|------|--------|
| "Component validation failed" | 無効なコンポーネントプロパティ | プロパティ値と範囲を確認 |
| "Connection not allowed" | 互換性のないコンポーネントタイプ | 接続ルールを確認 |
| "OpenWAM execution failed" | シミュレーションエラー | 入力ファイルとログを確認 |
| "Database connection failed" | SQLiteの問題 | ファイル権限を確認 |
| "File upload failed" | ファイルサイズ/形式の問題 | ファイル制約を確認 |

### OpenWAMエラー

| エラーメッセージ | 原因 | 対処法 |
|-----------------|------|--------|
| "Error reading input file" | ファイル形式エラー | .wamファイル構文を確認 |
| "Convergence not achieved" | 数値不安定性 | 時間刻みまたはメッシュを調整 |
| "Invalid boundary condition" | BC設定エラー | 境界設定を確認 |
| "Memory allocation failed" | メモリ不足 | モデル複雑さを削減 |

## モデル作成・編集の問題

### コンポーネントが配置できない

**症状**: コンポーネントパレットからコンポーネントを選択してもキャンバスに配置できない

**対処法**:

1. **ブラウザの互換性確認**
   - 対応ブラウザ（Chrome, Firefox, Safari, Edge）を使用
   - JavaScriptが有効になっているか確認

2. **キャンバスの状態確認**
   - キャンバスが正しく初期化されているか確認
   - ページを再読み込み

3. **マウス操作の確認**
   - コンポーネント選択後、キャンバス上でクリック
   - ドラッグ&ドロップではなくクリックで配置

### コンポーネントの接続ができない

**症状**: 接続ツールを使ってもコンポーネント同士が接続されない

**対処法**:

1. **接続ルールの確認**
   - 接続可能なコンポーネントの組み合わせか確認
   - [コンポーネントリファレンス](component-reference.md)で接続ルールを確認

2. **接続点の確認**
   - 正しい接続点をクリックしているか確認
   - 接続点が緑色に変化することを確認

3. **接続順序の確認**
   - 出力側から入力側の順序で接続
   - 双方向接続の場合は順序は問わない

### プロパティが保存されない

**症状**: コンポーネントのプロパティを変更しても保存されない

**対処法**:

1. **入力値の検証**
   - 入力値が有効な範囲内か確認
   - エラーメッセージが表示されていないか確認

2. **フォーカスの移動**
   - 入力後、他の要素をクリックしてフォーカスを移動
   - EnterキーまたはTabキーで入力を確定

3. **プロジェクトの保存**
   - Ctrl+Sでプロジェクトを明示的に保存
   - 自動保存が有効になっているか確認

## シミュレーション実行の問題

### シミュレーションが開始されない

**症状**: 「シミュレーション実行」ボタンを押してもシミュレーションが開始されない

**対処法**:

1. **モデルの検証**
   - モデル検証を実行してエラーがないか確認
   - 必要な境界条件が設定されているか確認

2. **OpenWAM実行ファイルの確認**
   - OpenWAM実行ファイルが正しいパスに配置されているか確認
   - 実行権限が設定されているか確認

3. **入力ファイルの生成**
   - WAMファイルが正しく生成されているか確認
   - ファイルの書き込み権限があるか確認

### シミュレーションが途中で停止する

**症状**: シミュレーションが開始されるが途中で停止してしまう

**対処法**:

1. **収束問題**
   ```
   エラー: Convergence failed at time step X
   ```
   - 時間刻み（角度増分）を小さくする
   - 初期条件を調整する
   - 境界条件の妥当性を確認

2. **メモリ不足**
   ```
   エラー: Out of memory
   ```
   - 計算セル数を減らす
   - 不要なコンポーネントを削除
   - システムメモリを増設

3. **数値不安定性**
   ```
   エラー: Numerical instability detected
   ```
   - CFL条件を満たすよう時間刻みを調整
   - メッシュサイズを見直す
   - 境界条件の急激な変化を緩和

### 結果が非物理的

**症状**: シミュレーション結果が物理的に不合理

**対処法**:

1. **パラメータの確認**
   - 入力パラメータの単位と値を確認
   - 物理的に妥当な範囲内か確認

2. **モデル構造の見直し**
   - コンポーネントの接続が正しいか確認
   - 境界条件の設定が適切か確認

3. **初期条件の調整**
   - 初期圧力・温度が適切か確認
   - 初期流速が妥当か確認

## ファイル操作の問題

### ファイルのアップロードができない

**症状**: WAMファイルをアップロードしようとするとエラーが発生

**対処法**:

1. **ファイル形式の確認**
   - ファイル拡張子が.wamか確認
   - ファイルの内容が正しいWAM形式か確認

2. **ファイルサイズの確認**
   - ファイルサイズが制限（10MB）以下か確認
   - 不要なデータを削除してサイズを削減

3. **ファイルの権限確認**
   - ファイルが読み取り可能か確認
   - 他のアプリケーションで使用中でないか確認

### ファイルのダウンロードができない

**症状**: 結果ファイルやモデルファイルのダウンロードが失敗する

**対処法**:

1. **ブラウザの設定確認**
   - ダウンロード先フォルダの権限を確認
   - ポップアップブロッカーを無効化

2. **ファイルの存在確認**
   - サーバー上にファイルが存在するか確認
   - ファイル生成が完了しているか確認

3. **ネットワーク接続確認**
   - ローカルネットワークの接続を確認
   - ファイアウォールの設定を確認

### WAMファイルの解析エラー

**症状**: アップロードしたWAMファイルの解析に失敗する

**対処法**:

1. **ファイル形式の確認**
   ```
   エラー: Invalid WAM file format
   ```
   - OpenWAM標準形式に準拠しているか確認
   - 文字エンコーディングを確認（UTF-8推奨）

2. **ファイル内容の確認**
   ```
   エラー: Missing required sections
   ```
   - 必要なセクションが全て含まれているか確認
   - データの整合性を確認

3. **バージョン互換性**
   ```
   エラー: Unsupported OpenWAM version
   ```
   - 対応するOpenWAMバージョンか確認
   - 必要に応じてファイルを変換

## パフォーマンスの問題

### アプリケーションの動作が重い

**症状**: ユーザーインターフェースの応答が遅い

**対処法**:

1. **ブラウザの最適化**
   - 不要なタブを閉じる
   - ブラウザを再起動
   - ハードウェアアクセラレーションを有効化

2. **モデルの最適化**
   - 不要なコンポーネントを削除
   - 計算セル数を適切に設定
   - 複雑な接続を簡略化

3. **システムリソース**
   - メモリ使用量を確認
   - CPUの負荷を確認
   - 他のアプリケーションを終了

### シミュレーションが遅い

**症状**: シミュレーション実行に非常に時間がかかる

**対処法**:

1. **計算条件の最適化**
   - 時間刻みを適切に設定
   - シミュレーション時間を短縮
   - 収束判定基準を調整

2. **並列計算の活用**
   - OpenMP並列化を有効化
   - CPUコア数に応じた設定
   - メモリ帯域幅の確認

3. **モデルの簡略化**
   - 不要な詳細を省略
   - 対称性を利用した簡略化
   - 境界条件の簡素化

## ブラウザ関連の問題

### 表示が崩れる

**症状**: ユーザーインターフェースのレイアウトが正しく表示されない

**対処法**:

1. **ブラウザの更新**
   - 最新バージョンに更新
   - 対応ブラウザを使用

2. **画面解像度の確認**
   - 推奨解像度（1280x720以上）を使用
   - ズームレベルを100%に設定

3. **CSS・JavaScriptの確認**
   - 開発者ツールでエラーを確認
   - キャッシュをクリア

### キーボードショートカットが効かない

**症状**: Ctrl+S等のショートカットが機能しない

**対処法**:

1. **フォーカスの確認**
   - アプリケーション画面にフォーカスがあるか確認
   - 入力フィールド以外をクリック

2. **ブラウザ設定の確認**
   - ブラウザのショートカットと競合していないか確認
   - 拡張機能の影響を確認

3. **代替操作の使用**
   - メニューボタンから同じ機能を実行
   - マウス操作で代替

## 問題解決のフローチャート

```
問題発生
    ↓
エラーメッセージの確認
    ↓
該当するカテゴリの確認
    ├─ システム関連 → システムエラーの対処法
    ├─ モデル関連 → モデルエラーの対処法
    ├─ シミュレーション関連 → シミュレーションエラーの対処法
    └─ ファイル関連 → ファイルエラーの対処法
    ↓
対処法の実行
    ↓
問題解決？
    ├─ Yes → 完了
    └─ No → 詳細ログの確認・サポートへ連絡
```

## ログファイルの確認方法

### アプリケーションログ

1. **ログファイルの場所**
   - `logs/combined.log`: 全般的なログ
   - `logs/error.log`: エラーログのみ

2. **ログレベル**
   - ERROR: エラー情報
   - WARN: 警告情報
   - INFO: 一般情報
   - DEBUG: デバッグ情報

3. **ログの読み方**
   ```
   [2024-01-01 10:00:00] ERROR: SIM001 - Convergence failed at time step 0.25
   ```
   - タイムスタンプ
   - ログレベル
   - エラーコード
   - 詳細メッセージ

### ブラウザの開発者ツール

1. **開発者ツールの開き方**
   - Chrome/Edge: F12キー
   - Firefox: F12キー
   - Safari: Cmd+Option+I

2. **コンソールタブの確認**
   - JavaScriptエラーの確認
   - ネットワークエラーの確認
   - 警告メッセージの確認

### 必要な情報

   - 関連するファイル
