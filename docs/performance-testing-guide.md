# パフォーマンステスト実行ガイド

## 概要

このドキュメントは、OpenWAM Engine Simulator Webアプリケーションのパフォーマンスと安定性テスト（タスク10.6）の実行方法と評価基準について説明します。

## テスト構成

### 実装されたテストケース

#### 1. 大規模モデル（50+コンポーネント）での操作性能テスト
- **目的**: 大量のコンポーネントを配置した際の操作性能を評価
- **テスト内容**:
  - 50個以上のコンポーネントを順次配置
  - 各コンポーネント配置時間の測定
  - 10コンポーネントごとのレンダリング性能測定
  - メモリ使用量の監視
- **評価基準**:
  - 平均コンポーネント配置時間: 1秒以内
  - 平均レンダリング時間: 0.5秒以内
  - 最低30コンポーネントの配置完了

#### 2. 長時間操作での安定性テスト（メモリリーク確認）
- **目的**: 長時間の連続操作でのメモリリークと安定性を評価
- **テスト内容**:
  - 5分間の連続操作実行
  - 30秒ごとのメモリ使用量監視
  - 様々な操作の繰り返し実行
- **評価基準**:
  - メモリ増加率: 50%以内
  - 最低100回の操作完了
  - システムクラッシュなし

#### 3. 複雑な接続を持つモデルでの描画性能テスト
- **目的**: 複雑な接続構造を持つモデルでの描画性能を評価
- **テスト内容**:
  - 複数カテゴリのコンポーネント配置
  - コンポーネント間接続の作成
  - ズーム・パン操作の性能測定
- **評価基準**:
  - 平均接続描画時間: 300ms以内
  - 平均ズーム描画時間: 100ms以内
  - 平均パン描画時間: 150ms以内

#### 4. ファイルI/O性能テスト（大容量ファイル処理）
- **目的**: 大容量ファイルの処理性能を評価
- **テスト内容**:
  - 異なるサイズのファイルアップロード
  - ファイル処理時間の測定
  - エクスポート機能の性能測定
- **評価基準**:
  - 平均アップロード時間: 5秒以内
  - 平均ダウンロード時間: 3秒以内
  - 平均処理時間: 2秒以内

#### 5. WebSocket通信安定性テスト（シミュレーション進行状況）
- **目的**: WebSocket通信の安定性とレイテンシを評価
- **テスト内容**:
  - WebSocket接続の監視
  - シミュレーション進行状況の更新確認
  - 通信エラーの記録
- **評価基準**:
  - 接続成功率: 80%以上
  - 平均レイテンシ: 1秒以内
  - 接続エラー数: 5回以内

#### 6. 総合パフォーマンス評価テスト
- **目的**: アプリケーション全体の総合的なパフォーマンスを評価
- **テスト内容**:
  - アプリケーション起動時間測定
  - ナビゲーション性能測定
  - 基本操作性能測定
  - パフォーマンススコア算出
- **評価基準**:
  - 起動時間: 10秒以内
  - 平均ナビゲーション時間: 0.5秒以内
  - 平均操作時間: 1秒以内
  - 総合スコア: 60点以上

## テスト実行方法

### 前提条件
1. アプリケーションサーバーが起動していること
2. Playwrightがインストールされていること
3. 十分なシステムリソースが利用可能であること

### 実行コマンド

#### 全テストの実行
```bash
npx playwright test e2e/ui-test-10.6-performance-stability.spec.ts
```

#### 特定のブラウザでの実行
```bash
npx playwright test e2e/ui-test-10.6-performance-stability.spec.ts --project=chromium
```

#### ヘッドレスモードでの実行
```bash
npx playwright test e2e/ui-test-10.6-performance-stability.spec.ts --headed
```

#### タイムアウト設定付きの実行
```bash
npx playwright test e2e/ui-test-10.6-performance-stability.spec.ts --timeout=300000
```

### 個別テストの実行

#### 大規模モデルテストのみ
```bash
npx playwright test e2e/ui-test-10.6-performance-stability.spec.ts -g "大規模モデル"
```

#### 長時間安定性テストのみ
```bash
npx playwright test e2e/ui-test-10.6-performance-stability.spec.ts -g "長時間操作"
```

#### WebSocket通信テストのみ
```bash
npx playwright test e2e/ui-test-10.6-performance-stability.spec.ts -g "WebSocket"
```

## パフォーマンス評価基準

### スコア算出方法

総合パフォーマンススコアは以下の要素から算出されます：

1. **起動時間評価（30点満点）**
   - 2秒以内: 満点
   - 3秒以内: -5点
   - 5秒以内: -10点
   - 5秒超過: -15点

2. **ナビゲーション時間評価（25点満点）**
   - 100ms以内: 満点
   - 200ms以内: -5点
   - 300ms以内: -10点
   - 300ms超過: -15点

3. **操作時間評価（25点満点）**
   - 300ms以内: 満点
   - 500ms以内: -5点
   - 800ms以内: -10点
   - 800ms超過: -15点

4. **メモリ使用量評価（20点満点）**
   - 100MB以内: 満点
   - 150MB以内: -5点
   - 200MB以内: -10点
   - 200MB超過: -15点

### 品質レベル分類

- **優秀（90-100点）**: 商用レベルの高性能
- **良好（70-89点）**: 実用レベルの性能
- **普通（50-69点）**: 基本的な性能要件を満たす
- **要改善（50点未満）**: パフォーマンス改善が必要

## トラブルシューティング

### よくある問題と対処法

#### 1. テストタイムアウト
**症状**: テストが時間内に完了しない
**対処法**:
- `--timeout`オプションでタイムアウト時間を延長
- システムリソースの確認
- 他のアプリケーションの終了

#### 2. メモリ不足エラー
**症状**: 大規模モデルテストでメモリエラー
**対処法**:
- 利用可能メモリの確認
- テスト対象コンポーネント数の削減
- ブラウザプロセスの再起動

#### 3. WebSocket接続エラー
**症状**: WebSocket通信テストが失敗
**対処法**:
- サーバーの起動状態確認
- ポート番号の確認
- ファイアウォール設定の確認

#### 4. 要素が見つからないエラー
**症状**: UI要素の取得に失敗
**対処法**:
- アプリケーションの完全起動を待機
- セレクターの確認
- 画面解像度の調整

## 結果の解釈
