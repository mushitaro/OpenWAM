# DS2 テレメトリ記録 (Stage 76)

Live (DS2) タブの「記録」がここに書き込みます。走行ログの受け渡し場所であり、
分析はこのフォルダのファイルを直接読んで行います(CSV ダウンロード不要)。

## 命名規約 = 出所

| ファイル名 | 中身 | git |
|---|---|---|
| `20260717_143000.json` | **実車** (K-Line / Web Serial) の計測データ | **追跡する**(走行しないと再取得できない一次データ) |
| `mock_20260717_143000.json` | モックリンクまたは self-check の合成データ | 追跡しない (.gitignore) |

`mock_` 接頭辞は、合成データを実測として読んでしまう事故を防ぐためのガードです。
接頭辞の有無だけで判別できるようにしてあります(`meta.source` にも記録)。

## 中身

```jsonc
{
  "log_id": "20260717_143000",
  "meta": {
    "created_at": "…", "source": "webserial", "vin": "…", "software": "…",
    "blocks": [3, 19, 35],
    "n_samples": 3182,
    "complete": true      // false = 記録中のチェックポイント (途中で切れた記録)
  },
  "samples": [ { "t": 0.21, "rpm": 3902, "rf": 118.4, "ro": 99.1, … } ]
}
```

サンプルの型は `frontend/lib/dme-link/types.ts` の `LiveSample`。
単位は ECU 生値のスケール済み値(rf = %、evan/avan = °KW、psau = mbar)。

## 保存の耐障害性

記録中は 20 秒ごとに同じ `log_id` へ上書きチェックポイント(`complete: false`)。
K-Line 断・タブクローズ・バックエンド停止でも、失うのは最後の 20 秒だけです。
停止時に `complete: true` で最終書き込み。保存に失敗した場合はタブ内にデータが
残るので、「再保存」ボタン(バックエンド復帰後)か CSV 退避を使ってください。

## 使い道

- Validation ビュー (`POST /validation/compare`) がマップ軸 (rpm × RO) にビン集計し、
  実測 rf とシミュ VE を同一 ECU 単位で比較します。
- 3900-5300 WOT 帯の「実測 − シミュ」は**箱モード寄与の実測値**として読みます
  (校正誤差ではない — 1D モデルは 3D 箱モードを持てない)。
